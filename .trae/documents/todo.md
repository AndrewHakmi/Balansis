### Этап 1: Оптимизация ядра (Low-Level Core)

1.  **Портирование на компилируемый язык:**
    *   Реализовать структуры `AbsoluteValue` и `EternalRatio` на Rust (с использованием PyO3) или C++ (с использованием pybind11).
    *   Перенести логику `Compensator` в нативный код для исключения оверхеда Python VM.
2.  **Векторизация операций:**
    *   Реализовать поддержку SIMD-инструкций (AVX2/AVX-512) для массовых операций сложения `⊕` и умножения `⊗`.
    *   Создать API для пакетной обработки данных (batch processing) вместо поэлементной.
3.  **Управление памятью:**
    *   Реализовать собственный аллокатор памяти (Memory Arena) для объектов `AbsoluteValue` для минимизации фрагментации кучи и нагрузки на Garbage Collector.

### Этап 2: Интеграция с экосистемой Data Science

4.  **NumPy интеграция:**
    *   Создать пользовательский `dtype` для NumPy.
    *   Реализовать `ufuncs` (universal functions) для базовых арифметических операций, совместимые с NumPy API.
5.  **Pandas расширение:**
    *   Разработать `ExtensionDtype` и `ExtensionArray` для нативной поддержки колонок типа `AbsoluteValue` в Pandas DataFrame.
    *   Переопределить методы агрегации (`sum`, `mean`, `std`) для работы с логикой `EternalRatio` (игнорирование сингулярностей).
6.  **Сериализация:**
    *   [x] Реализовать поддержку протокола Apache Arrow для эффективной передачи данных между системами.
    *   [x] Написать методы `__reduce__`/`__getstate__`/`__setstate__` для поддержки `pickle` и сохранения состояния объектов.

### Этап 3: Расширенная математика и Алгоритмы

7.  **Трансцендентные функции:**
    *   [x] Реализовать функции `log`, `exp`, `sin`, `cos`, `tan` для аргументов типа `AbsoluteValue` и `EternalRatio`.
    *   [x] Определить поведение этих функций в точках сингулярности (0): `log(Absolute)` запрещён; `sin(0)=0`, `cos(0)=1`, `tan(0)=0`.
8.  **Линейная алгебра:**
    *   [x] Базовая реализация матричного умножения (GEMM) с компенсирующим сложением.
    *   Реализовать алгоритмы разложения матриц (SVD, QR), устойчивые к делению на ноль.

### Этап 4: Специализированные модули

9.  **ML / Deep Learning адаптер:**
    *   Создать кастомный `Optimizer` для PyTorch (наследник от `torch.optim.Optimizer`), реализующий обновление весов через `EternalRatio`.
    *   Написать кастомные слои (Layers), использующие ACT-арифметику для Forward и Backward pass.
10. **Финансовый модуль:**
    *   Реализовать класс `Ledger` для строгого сведения баланса (Double-entry bookkeeping) с гарантией нулевой суммы.
    *   Добавить поддержку фиксированной точности (Fixed-point arithmetic) поверх `AbsoluteValue`.

### Этап 5: Тестирование и Верификация

11. **Сравнительные бенчмарки:**
    *   Создать набор тестов производительности (vs `float`, vs `decimal`, vs `numpy`) для базовых операций и матричных вычислений.
12. **Формальная верификация:**
    *   Описать аксиомы ACT на языке Coq или Lean.
    *   Автоматически доказать непротиворечивость базовых операций (`⊕`, `⊗`) и свойств группы/поля.
13. **Fuzzing-тестирование:**
    *   Настроить `Hypothesis` или `AFL` для генерации случайных последовательностей операций с целью поиска краевых случаев, нарушающих ассоциативность или коммутативность.
